# 方法学与技术基础

本文说明 CaseLaw-Bench(买卖合同·上海·v1.0)的构建方法、设计依据,以及用于自证可靠的证据。关于如何使用,见 README;关于版本变更,见 CHANGELOG。

## 1. 设计取向

本基准采用分级相关性(0–3),而非二元的"是否类案"。分级保留了程度信息,既能计算 NDCG,也能向下折叠为二元或三元标注,补足了既有民商事评测(如 CAIL-SCM)只测相对排序、不测绝对分级召回的缺口。

相关性判断采用 pointwise,即每个查询-候选对单独打分,而不是让模型一次性对整库排序(listwise)。分级标准答案并不需要全序,而 listwise 在法律长文本上既不稳定、延迟也高;caselaw-retrieval 的实测也印证了这一点。

判级的实质标准是"实质相似优先于形式相似"。相似性落在决定裁判规则形成的关键性事实上,而非话题、标签或文书模板的表面。两案即便措辞高度雷同,只要决定裁判规则的关键不同点占据压倒分量,即不构成类案。这一标准取自最高人民法院《指导意见》的三步法,也是本基准区别于纯文本相似度的根本所在。

最后,溯源与置信分开记录。每条标注都附带判官模型、prompt 版本、双模型分值、不确定度和校验状态;这些记录回答"依据是什么",与"判得有多准"是两件事。

## 2. 依据的标准与文献

### 类案的定义

相关性标准直接采用最高人民法院《关于统一法律适用加强类案检索的指导意见(试行)》,而非另立标准。该意见将类案界定为在基本事实、争议焦点、法律适用三方面与待决案件相似的生效案件,并给出"固定异同—聚焦关键性事实—权衡何者对裁判规则更具压倒分量"的三步识别法。我们抽取的结构化字段(案情核心、法律争点、裁判要旨及所引法条)正对应这三个要素,因此判级理由可逐案沿三步法展开。意见同时要求比对前先查生效,据此设有效力门槛(改判、发回、法条失效者排除)。

### 用大模型做分级相关性判断

让大模型做相关性判断在信息检索评测中已有较多验证。UMBRELA(TREC RAG 2024 采用)报告分级 pointwise(0–3)与人工判断的相关性可达 τ>0.87,在各类大模型评估方法中系统级排序一致性最强。Thomas 等人以及 SIGIR 2024 的法律案件检索工作进一步表明,大模型相关性判断可用于离线评测,且"先抽取事实再判断相关性"优于让模型直接通读全文——后者容易被表面细节带偏。本基准正是先做结构化抽取,再把结构化要素交给判官。

在分级的粒度上,我们不让模型输出自由连续分。Zhuang 等人的工作表明,超过四档或使用自由连续值会带来虚假精度,三档与四档、0–4 与 0–8 的效果并无实质差别。因此每次调用只产离散的 0–3,连续分通过对离散档求期望得到,而非由模型凭空给出。

连续分与置信度则来自自我一致性采样。已有工作显示多次采样后聚合能提升排序质量,且采样方差本身就是一个天然的置信信号。据此,每对查询-候选独立判 K=10 次,以均值作连续金标、以标准差作不确定度。

需要说明,相关性判断本身带有主观性。LeCaRD 与 LeCaRDv2 报告法律专家之间的 κ 约为 0.50–0.52,这大致是这类判断一致性的上限,人或机器都难以远超。本基准的跨模型加权 κ 为 0.59,即对标这一量级。

## 3. 构建流程

构建分为抽取、谱系、选题、打分四个阶段。

抽取分两层。规则层用正则处理确定性字段——案号、法院、审级、段落切分、法条、当事人,实测命中率在 99% 以上;只有需要语义理解的字段(争议焦点评析、法律关系、标的物、裁判结果归类)才交给大模型(deepseek-v4-flash)。这样既控制成本,也避免在可规则化的字段上引入模型噪声。最终得到 751 篇结构化案件。

在结构化案件之上,我们建立了两级争议焦点谱系。受控词表提供粗标签,用于 Recall 层面的粗匹配;但在单一案由的库内,"货款给付"一类粗标签覆盖近八成案件,会把数百篇性质各异的案件压成一类。真正的判别力在法律争点和裁判要旨所蕴含的、裁判规则层面的细争点。我们用大模型先归纳、再归类,得到 53 个叶子、七层的细谱系,其判据是"同一叶子近似对应同一裁判规则"。实测约 68% 的案件交织两到三个细争点,最大的主叶子也只占 14%。

查询的选取以判别力为先。配额按反比例分配,压低货款类多数、保留稀有争点,并刻意纳入 19 个表里不一的陷阱案——这些案件焦点标签看似货款,裁判规则实则落在股东连带、保证责任等其他法域,用以考察检索系统是否会被表面相似误导。选定后,每个案件被改写成一句中立第三人称、泛化脱去具体信息、只问不答的自然语言问句,以匹配下游检索系统的真实输入。改写还有一个副产物:问句的来源案保留在候选池内,自然成为该问句的高相关锚点,补上了去重后失去的 sanity 正例。

打分阶段,判官读问句与候选案件的结构化要素,沿三步法和效力门槛产出离散 grade。判官选用 DeepSeek 系模型,与被测检索/精排系统所用模型不同,以避免循环偏倚。

## 4. 评分与聚合

每对查询-候选判 K=10 次(温度约 0.9),聚合为三个量:grade_主档(均值四舍五入,离散 0–3,用于 Recall/MRR/MAP 的阈值与一致性计算)、grade_期望(均值,连续 0–3,作 NDCG 的增益,边界处按比例给分)、grade_std(标准差,反映不确定度,值大者进入人工复核队列)。

温度的取值经过校准。温度 0.5 时十次采样几乎没有方差,连续分退化为整数,等同于只判一次;温度提到 0.8 以上,清晰的案件仍稳定在同一档,而边界案件开始出现有意义的方差,连续分辨率正是由此而来。这一观察确定了 0.8–1.0 的取值区间。

为控制计算量,打分分两阶段。第一阶段对全部候选各判一次做粗筛;第二阶段只对粗筛 grade≥1 的对补满十次。明显不相关的异叶子候选在单次判断下已经相当稳定,不必反复采样。43,558 个候选对最终消耗约 28.8 万次调用。

## 5. 质量验证

可靠性证据来自四个方面。

第一是跨模型一致性。用第二个、且更强的模型 deepseek-v4-pro 重判 150 个各档均衡的锚样本,与主判官 deepseek-chat 比较:加权 κ 为 0.59,相邻一档以内的一致率为 93%,连续分的 Pearson 相关为 0.65,大致相当于人际水平。需要诚实指出,未加权 κ 仅 0.256,混淆矩阵显示 v4-pro 系统性地更严,把主判官的 grade-2 大量压到 grade-1。这意味着"类案"的绝对阈值依赖于判官标定,其中 grade-2 这条边界最不稳定,grade-3 则相当稳固。

针对这一点,我们做了双判官集成。对全部类案候选(grade≥2)用 v4-pro 复判,连续分取两个模型的均值。类案数量从单判官的 2,369 收紧到双模型共识的 1,370,约 42% 的"软类案"被剔除;其中 grade-3 由 466 变为 351,没有一个跌出类案范围。

第三是基准的区分度,也即这把尺子能否区分系统的好坏。BM25 基线在留一法评测下,NDCG@10(不含锚点)约为 0.51(索引原文段落)到 0.61(索引抽取出的焦点评析),而随机基线约 0.24、理想上界为 1.0。BM25 落在两者之间,说明标准答案确有信号、任务又不至于平凡,并为语义检索留出约 0.4 的提升空间。原文与焦点评析两版相差约 0.10,反映干净抽取带来的字面增益;但二者距上界仍有约 0.4,说明标准答案奖励的是超出词面的实质相似——纯字面匹配难以应对陷阱案。

第四是锚点自检与复测一致性。58 个查询中,有 54 个的来源案连续分不低于 2.5;其余四个偏低,逐一核对后确认是真实的法理差别(少数说、稀有薄叶子),而非标注错误。集成后连续分的组内相关约 0.41,因纳入了跨模型分歧而较单判官时更低,这是对真实不确定性的诚实反映;离散主档则更稳定。

## 6. 可复现性

| 环节 | 配置 |

| 抽取 | deepseek-v4-flash,prompt 版本 extract-v4 |
| 细争点谱系 | fine-v0.2,53 叶子 |
| 主判官 | deepseek-chat,prompt 版本 judge-v2,K=10,温度 0.9 |
| 集成判官 | deepseek-v4-pro,K=3,连续分取均值 |

每条标注在 qrels.explained.jsonl 中保留三步法理由、判官模型、prompt 版本与双模型分值。qrels 一经冻结即不修改;后续新增或重标另开版本,评测报告须注明所依据的版本。

## 7. 局限

数据限于上海、买卖合同纠纷、2022 年,结论不外推到其他地域或案由。当前的可靠性证据是两个大模型之间的一致,而非对法律真值的人工校验;完整的人工 κ 尚未做,但 150 对锚样本已备好,可随时补充。类案的绝对数量依赖判官标定,更严的模型会判出更少类案,因此排序和连续分比绝对的类案计数更稳健,建议优先采信 NDCG。单一案由库内 grade-1(同属买卖框架的弱相关)数量很大,有效刻度实际上是 1 到 3;争点较宽的查询类案偏多。效力门槛在本阶段做了简化,未单独合成效力字段;新旧法混引(民法典与合同法)的逐条映射列为后续工作。
