# CaseLaw-Bench:买卖合同(上海)· 民商事类案检索分级相关性基准 v1.0

面向中文民商事类案检索的分级相关性评测基准。Phase 1 为买卖合同纠纷,上海。输入一句自然语言法律问句,标注库内每篇案件与之的类案相关性等级,用于评测检索系统。

| 项 | 内容 |
|---|---|
| 版本 | v1.0(2026-06-11) |
| 任务 | 类案检索 / Legal Case Retrieval,分级相关性 0–3 |
| 语言与法域 | 中文,中国大陆民商事;Phase 1 限上海·买卖合同纠纷·2022 |
| 规模 | 58 查询 · 751 候选案件 · 43,558 个已判对 |
| 标注 | 大模型双判官集成(deepseek-chat × deepseek-v4-pro),pointwise + 自我一致性 |
| 指标 | NDCG@10 · Recall@20 · MRR · MAP |
| 许可 | 见 LICENSE(源文书公开,标注供研究用途) |

## 这是什么

这是给类案检索系统用的评测集,不是检索引擎。给定一条查询(一句自然语言法律问句),标准答案标注了库内每篇案件与该问句的类案相关性等级(0–3),据此可计算检索系统的 NDCG@10、Recall@20、MRR、MAP。

民商事类案检索长期缺少自有的分级相关性标准答案,此前只能借用 CAIL-SCM(民间借贷,二元)或 LeCaRD(刑事,降级使用)。本基准填补这一空缺。

相关性的判据取自最高人民法院《关于统一法律适用加强类案检索的指导意见》:两案在基本事实、争议焦点、法律适用(裁判规则)三方面是否实质相似。核心标准是实质相似优先于形式相似——话题或标签再像,只要决定裁判规则的关键不同点占据压倒分量,即不构成类案。方法学与文献依据见 METHODOLOGY.md。

## 数据格式

corpus.jsonl,每行一篇案件:

```jsonc
{
  "doc_id": "(2021)沪0104民初28195号#b6df02",   // 唯一键 = 归一化案号 + 内容指纹
  "案号": "...", "法院": "...", "审级": "一审", "案由": "买卖合同纠纷", "裁判日期": "2022-09-27",
  "当事人": [{"角色":"原告","名称":"...","类型":"企业"}, ...],
  "法律关系": "普通买卖", "标的物类型": "服务器",
  "争议焦点": {"焦点标签":[...], "焦点评析":{"案情核心":"...","法律争点":"...","裁判要旨":"..."}, "是否显式": false},
  "细争点": {"主叶子":"D1_一人公司股东财产混同连带", "细争点":[...], "裁判规则争点":"..."},
  "裁判结果_标签": "全部支持", "引用法条": [{"法律":"公司法","条":63}, ...],
  "段落": {"诉称":"...","查明事实":"...","本院认为":"...","裁判结果":"..."}   // 去模板的实体文本,供检索
}
```

queries.jsonl,每行一条查询:

```jsonc
{
  "query_id": "Q-015",
  "query_text": "一人公司的买受人出具欠款证明后，其原股东未能证明财产独立，是否应对公司债务承担连带责任？",
  "query_source_doc": "(2021)沪0104民初28195号#b6df02",  // 问句出处，留在候选池内，作高相关锚点
  "主叶子": "B4_对账单欠条证明力", "难度": "hard", "陷阱": true
}
```

qrels.jsonl,每行一条查询的标准答案;未列出的候选默认 grade 0:

```jsonc
{
  "query_id": "Q-015",
  "query_source_doc": "...", "query_text": "...", "主叶子": "...",
  "candidates": [
    {
      "doc_id": "...",
      "grade_主档": 3,          // 离散 0–3，Recall/MRR/MAP 取 ≥2 阈值
      "grade_期望": 2.9,        // 连续 0–3，NDCG 的增益（双模型均值）
      "grade_std": 0.21,        // 不确定度（K 次采样 + 跨模型分歧）
      "grade_期望_chat": 3.0,   // 主判官 deepseek-chat
      "grade_期望_v4pro": 2.8   // 交叉判官 deepseek-v4-pro
    }
  ]
}
```

完整版 qrels.explained.jsonl 另含每条的三步法理由、对齐焦点、效力与置信。

## 评测你的系统

先让检索系统对每条查询输出对全库 751 篇的排序,写成 run 文件,每行一条:

```jsonc
{"query_id": "Q-015", "ranking": ["doc_id_1", "doc_id_2", "..."]}
```

再运行评测(无第三方依赖):

```bash
python eval.py --qrels data/qrels.jsonl --run your_run.jsonl
```

输出 NDCG@10、Recall@20、MRR、MAP,分含锚点与不含锚点两版,按查询宏平均。也可用 TREC 生态:`trec_eval data/qrels.trec.txt your_run.trec`。

口径上,NDCG 用连续的 grade_期望;Recall@20、MRR、MAP 用 grade_主档≥2 作二值正例。单一案由库内 grade 1(同属买卖框架的弱相关)很多,有效刻度实际上是 1 到 3,故以 grade≥2 为类案阈值。无正例的查询按 IR 惯例不计入二值指标。

## 相关性分级标准

过效力门槛后,按实质相似程度定档:

| grade | 名称 | 判据 |
|---|---|---|
| 3 | 高度类案 | 法律争点相同,决定裁判规则的关键事实相同点压倒不同点,且法律关系相同,可直接援引 |
| 2 | 类案 | 法律争点与法律关系相同,但部分裁判规则关键要素不同(标的、交易结构、抗辩有别),需类比调整 |
| 1 | 弱相关 | 争议焦点不同,仅同属买卖框架;或形式相似但关键不同点压倒,实质不相似 |
| 0 | 不相关或失效 | 真实法律关系或法域不同(名为买卖实为借贷,或争点被股东连带等他域压倒);或未过效力门槛(二审改判、发回、法条失效) |

## 基线成绩

58 查询 × 751 候选,留一法,按查询宏平均。完整数据见 baselines/leaderboard.md。

| 系统 | NDCG@10 | Recall@20 | MRR | MAP |
|---|---|---|---|---|
| random（下界） | 0.24 | 0.03 | 0.10 | 0.04 |
| BM25（原文段落） | 0.61 | 0.46 | 0.78 | 0.43 |
| BM25（焦点评析） | 0.72 | 0.51 | 0.96 | 0.53 |
| oracle（上界） | 1.00 | 0.82 | 1.00 | 1.00 |

BM25 落在随机与理想之间,说明标准答案有信号、任务又不平凡,并为语义检索留出空间。其与上界的差距,正是纯字面匹配难以应对的实质相似(例如焦点标签像货款、裁判规则实为股东连带的陷阱案)。下一步可接语义检索或精排,与此基线对比。

## 构建方法

完整说明见 METHODOLOGY.md,此处概述。判决书先经规则层抽取(案号、法院、审级、段落、法条,正则命中率 99% 以上)和大模型层抽取(争议焦点评析、法律关系、标的、裁判结果),得到 751 篇结构化案件;在此之上建立 53 叶子的裁判规则级争议焦点细谱系;按主叶子分层、判别力优先选出查询,并改写成自然语言问句;以 judge-v2(问句对案件)逐对打分,每对自我一致性十次;最后对类案带用 deepseek-v4-pro 复判取均值,得到 v1.0。判官与被测系统不使用同一模型,以避免循环偏倚;每条标注带溯源。

## 局限与范围

数据限于上海、买卖合同纠纷、2022 年,结论不外推到其他地域或案由。当前的可靠性证据是两个大模型之间的一致(跨模型加权 κ 为 0.59,相邻一档以内一致率 93%,连续分相关 0.65,大致相当于人际水平),而非对法律真值的人工校验;完整人工 κ 尚未做,150 对锚样本已备好可随时补充。类案的绝对数量依赖判官标定,更严的模型会判出更少类案,因此排序与连续分比绝对类案计数更稳健,建议优先采信 NDCG。grade_std 较大的对是判断最不确定处,可作人工复核的优先队列。源数据为脱敏公开判决,已去重;标注为大模型初标加自我一致性聚合再加双判官集成,非司法认定,不构成法律意见。
